# 7天启动计划：AI4S 电力市场交易赛道

## 计划基准

- 制定日期：2026-04-23
- 当前阶段：赛前启动与认知搭建
- 核心目标：在 7 天内形成“可讨论、可验证、可继续扩展”的 baseline 方案，而不是直接冲复杂模型

## Day 1：锁定题目边界

目标：

- 把赛题拆成清晰任务
- 固化当前认知
- 准备后续分工

动作：

- 阅读并维护 [长期记忆](/D:/TBK/AI/game/docs/长期记忆_电力市场交易赛道.md)
- 报名后第一时间下载并整理官方文档
- 从官方文档中优先确认：
  - 评分公式
  - 时间粒度
  - 预测目标
  - 储能约束
  - 提交格式
- 与项目经理完成第一次范围对齐

产出：

- 一页赛题拆解
- 一页待确认问题清单

## Day 2：建立业务最小模型

目标：

- 不依赖复杂算法，先把业务逻辑跑通

动作：

- 明确储能调度变量、约束和收益函数
- 写出纸面版优化问题
- 形成一个极简示意流程：
  - 输入未来价格
  - 输出充放电决策
  - 计算收益

产出：

- 储能优化问题说明
- 变量和约束表

## Day 3：建立数据与回测框架

目标：

- 先搭架子，防止后续反复返工

动作：

- 设计数据目录结构
- 设计训练集、验证集、测试集的时间切分方案
- 设计 rolling backtest
- 明确防泄漏规则

产出：

- 数据处理规范
- 回测规范

## Day 4：做价格预测 baseline

目标：

- 用最稳的方法快速跑出第一版

动作：

- 先尝试树模型：
  - LightGBM
  - CatBoost
  - XGBoost
- 先做基础特征：
  - 时间特征
  - 滞后特征
  - 滚动统计特征
  - 天气特征
  - 负荷与新能源相关特征

产出：

- 第一版价格预测 baseline
- 基础误差指标

## Day 5：接入储能优化与收益评估

目标：

- 让模型误差与最终收益形成闭环

动作：

- 将 Day 4 的价格预测接入调度器
- 对比：
  - 预测误差
  - 最终收益
- 构造 oracle 基线：
  - 使用真实未来价格求理论最优收益

产出：

- baseline 收益
- oracle 收益
- 收益 gap

## Day 6：业务校准

目标：

- 尽早纠偏业务假设

动作：

- 与山东虚拟电厂相关人士沟通
- 按固定问题清单确认：
  - 关键驱动因素
  - 常见业务误区
  - 约束是否遗漏
  - 是否存在典型尖峰或极端场景

产出：

- 业务校准纪要
- 假设修正清单

## Day 7：形成第一阶段结论

目标：

- 让项目进入稳定迭代状态

动作：

- 复盘第一版结果
- 识别最影响收益的误差来源
- 决定第二阶段重点是：
  - 特征增强
  - 模型增强
  - 优化器增强
  - 极端场景处理
- 规划下一周工作

产出：

- 第一阶段复盘
- 第二阶段路线图

## 每天固定检查项

- 是否有数据泄漏风险
- 是否以收益为主，而不是只盯预测误差
- 是否已有可复现结果
- 是否有未确认的业务假设
- 是否有提交链路风险

## 第一阶段验收标准

7 天结束时，至少应达到：

- 能清楚解释赛题三段结构
- 能清楚解释储能约束
- 有一个能跑通的价格预测 baseline
- 有一个能跑通的储能调度器
- 有一个 rolling backtest
- 有 oracle 对照
- 有面向后续 AI 的长期记忆文档

## 当前优先级排序

1. 官方文档确认
2. 业务约束理解
3. 回测正确性
4. baseline 可运行
5. 收益闭环
6. 外部业务校准
7. 复杂模型探索
