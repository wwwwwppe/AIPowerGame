# 长期记忆：AI4S 电力市场交易赛道

## 1. 项目背景

- 比赛：第四届世界科学智能大赛
- 赛道：赛道一，电力市场交易赛道
- 题目方向：储能电站收益优化
- 当前记录日期：2026-04-23
- 参赛目标：冲击冠军
- 当前组合：用户本人 + 公司项目经理
- 团队背景：公司主营 ERP，对电力市场和储能业务几乎没有实战经验，但 AI 应用、算法实现、调研与学习能力强
- 外部支持线索：山东一支做虚拟电厂系统的团队中，有人可能提供业务支持

## 2. 当前问题定义

这个题不能只理解成“电价预测”，而应该理解成一个三段式问题：

1. 理解电力现货市场与储能运行约束
2. 预测未来节点实时电价
3. 在储能约束下制定充放电策略，使最终收益最大化

核心结论：

- 这是“预测服务决策”的题，不是普通回归题
- 只卷预测误差，未必能赢
- 真正重要的是收益导向的建模、回测和优化闭环

## 4. 这道题真正考什么

### 4.1 业务层

需要理解以下概念，而不是只停留在名词层面：

- 电力现货市场
- 节点电价
- 日前市场与实时市场
- 负荷
- 新能源出力
- 气象影响
- 价差
- 电网拥塞与网络约束
- 储能作为灵活性资源的套利逻辑

### 4.2 模型层

关键不只是预测均值，而是：

- 抓住峰谷与尖峰
- 抓住高收益时段
- 控制关键时段的方向性错误
- 理解误差如何影响收益

### 4.3 优化层

必须把储能调度写成一个严谨的约束优化问题，至少要包含：

- SoC 递推
- 充放电功率上限
- 容量上限
- 充放电效率
- 初始 SoC
- 终端 SoC
- 不允许同时充放电的约束表达方式

### 4.4 工程层

很多队不是模型差，而是：

- 泄漏了
- 回测错了
- 提交挂了
- 结果不可复现

## 5. 当前推荐路线

## 5.1 总体路线

先走稳健的“两阶段”方案：

`价格预测或场景生成 -> 带约束的储能优化`

不建议第一阶段就直接上强化学习。

## 5.2 第一版 baseline

第一版应该尽快做出：

- 电价预测 baseline
  - LightGBM / CatBoost / XGBoost 三选一或多选
- 储能调度 baseline
  - 线性规划或凸优化
- 滚动回测
- 收益评估

## 5.3 为什么不先上复杂模型

当前团队短板在业务和约束，不在模型花样。

在没有锁定：

- 数据字段
- 评分方式
- 回测逻辑
- 提交格式

之前，上复杂深度学习和 RL 的投入产出比偏低。

- 目标

建议固定使用以下术语：

- rolling backtest

## 9. 必须向山东业务支持方确认的问题

建议第一次沟通重点问以下问题：

- 蒙西实时价格最核心的驱动因素有哪些，重要性如何排序
- 储能收益中最容易被忽略的约束是什么
- 更值得抓的是高价尖峰，还是低价或负价时段
- 节点价差更多来自新能源波动，还是网络拥塞
- 若只能保留少量特征，最值得优先看的是什么
- 没做过电力业务的团队最常犯的错误是什么

## 10. 第一阶段最应该产出的东西

不是论文式创新，而是：

- 一个无泄漏的数据处理管道
- 一个稳健的价格预测 baseline
- 一个可解释的储能调度器
- 一个滚动回测框架
- 一套收益分析图表
- 一个稳定可提交的工程包装

## 11. 当前不建议过早投入的方向

在拿到官方数据和评分细节前，不建议优先投入：

- 强化学习
- 多智能体
- 复杂 GNN
- 特别重的 Transformer 结构
- 高级潮流建模
- 大量论文式方法堆叠

## 12. 下一步行动原则

当前阶段的原则是：

- 先做稳，而不是先做炫
- 先锁业务约束，再卷模型
- 先把收益闭环跑通，再追求模型上限
- 先保证提交成功，再追求排行榜波动

## 13. 风险清单

当前已识别风险：

- 把题目误做成纯预测赛
- 用随机切分造成数据泄漏
- 预测指标好但收益差
- 储能约束写错
- 业务理解偏差无人校准
- 提交容器不稳定
- 过早进入复杂模型，拖慢进度

## 14. 后续所有 AI 会话建议遵循的工作顺序

后续不管是继续问当前 AI，还是换其他 AI 工具，建议都先按这个顺序推进：

1. 先读取本文件
2. 再读取 7 天启动计划
3. 明确当前卡点属于：
   - 业务理解
   - 数据理解
   - 建模
   - 优化
   - 回测
   - 工程交付
4. 只在当前卡点上向前推进，不随意发散
